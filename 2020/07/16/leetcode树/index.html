1. 递归

1.1 二叉树的最大深度(104)

  • 思路：

    • 假设root==null，return 0
    • 否则返回最大的左右子树的深度+1

1.2 平衡二叉树（110）

  • 思路：

    既然我们已经可以求出一棵树的深度，那么，就在root有左右子树的情况下判断深度即可

    但是这样却也不够，因为我们要的是每个节点的左右子树都必须如此，所以还必须&&

1.3 二叉树的直径（543）

  • 思考：

    树的直径一定属于下列三种情况：

    1. 左子树的直径
    2. 右子树的直径
    3. 穿越此节点的直径

    三者取最大值

    那么，穿越此节点的直径应该怎么取呢？

    最大直径就是左子树的深度+ 右子树的深度+1

1.4 翻转树（226）

  • 思考：左右翻转即可

1.5 归并两棵树（617）

  • 思路：

    • 如果左右节点都已经叠加好了，那么我们就只需要将两个根节点叠加并重新挂载左右节点即可

1.6 路径总和（112）

  • 思路：

    • 先从左节点考虑，依次向下寻找，终止条件为到达根节点&&sum==0，否则返回上一节点（dfs）

    这样的话我们就是在一直改变sum的值，递归回来的时候已经不是我们之前认识的sum了，所以最后要把它再+回来。

1.7 路径总和III（437）

  • 思考：
    • 这里我们的路径就不需要到叶子节点结束，也不需要从根节点开始。
    • 那么就有三种情况：
      • 从我们的根节点开始的
      • 不从我们的根节点开始的

1.8 另一个树的子树（572）

  • 思考：

    • 是子树的前提是子子孙孙都是一样的，而不能只是中间的一部分

      这里就有两种情况：以我为根（即必须和我一模一样），或者不以我为根，那也一定要和我的一个子孙一样

1.9 树的对称（101）

  • 思考：

    • 如果根节点为null，则返回真
    • 如果不是，则判断左右节点是否对称

1.11 左叶子之和（404）

  • 思路：

    • 假如只有根节点有值应当如何处理？

    • 结果应该是左子树的左叶子之和+ 右子树的左叶子之和

      假如当前节点的左节点为叶子节点，则加上对应的值，再加上当前节点右子节点的迭代

      假如不是，则返回当前节点的左右节点的迭代之和

1.12 最长同值路径

注意： 路径指的是边数，而不是节点数目

  • 思路：

    • 最长路径有三种情况：

      • 左子树上的最长路径
      • 右子树上的最长路径
      • 穿越根节点的最长同值路径

      所以我们需要一个辅助函数，用于返回穿越根节点的最长路径

      ​ 这个穿越根节点的最长路径又可以分成：左边/右边/左右都有

1.13 打家劫舍III

  • 思考：

    • 假设我们有一个辅助函数，能够找到以当前节点为根所能抢劫到的最大金额，那么整体的最大金额有三种情况：

      • 左边+ 右边的最大金额，且不加上根节点
      • 以当前节点为根的最大金额

    • 对于辅助函数

      • 应该等于左右子树的helper（）之和与root.val+下面两层的树的和的最大值

但是这样包含有许多的重复计算！！！

1.14 二叉树中第二小的节点

  • 思路：
    • 由于不会有一个结点的情况，并且有叶子节点的话，根节点就是较小的哪一个，所以可以推出，根节点一定是最小的，那么我们就是向左向右进行搜寻，直到找到较小的节点
    • 如果根节点的左右子树值均相等，则递归向下搜索，如果不等，则返回大的那一个
