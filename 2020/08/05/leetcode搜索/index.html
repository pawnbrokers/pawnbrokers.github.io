1. BFS

BFS 的核心思想应该不难理解的，就是把一些问题抽象成图，从一个点开始，向四周开始扩散。一般来说，我们写 BFS 算法都是用「队列」这种数据结构，每次将一个节点周围的所有节点加入队列。

BFS 相对 DFS 的最主要的区别是：BFS 找到的路径一定是最短的，但代价就是空间复杂度比 DFS 大很多。

1.1 应用场景

要说框架的话，我们先举例一下 BFS 出现的常见场景好吧，问题的本质就是让你在一幅「图」中找到从起点start到终点target的最近距离，这个例子听起来很枯燥，但是 BFS 算法问题其实都是在干这个事儿。

框架模板

队列q就不说了，BFS 的核心数据结构；cur.adj()泛指cur相邻的节点，比如说二维数组中，cur上下左右四面的位置就是相邻节点；visited的主要作用是防止走回头路，大部分时候都是必须的，但是像一般的二叉树结构，没有子节点到父节点的指针，不会走回头路就不需要visited。

1.2 二叉树的最小深度

  • 思路1：

    • 我们已经会求二叉树的深度了，它取得是左右子树的最大值

  • 思路2：

    • 采用BFS，广度优先搜索其实就是分层遍历，我们只要找到一层，其中有一个节点的左右子节点都是null了，也就意味着我们找到了最小深度

1.3 解开密码锁的最小次数（752）

  • 思路：

    • 我们首先想一下如何将所有的密码组合遍历出来：
    • 从0000开始，能够出现1000，9000，0100，0900。。。的8种组合，同理，每一步都能够衍生出八种组合

  • 既然能找到所有的密码组合，那么下一步我们就是判断终止条件，即找到target以及是否遇到了死亡密码

1.4 完全平方数（279）

  • 思路：

    • 动态规划，dp[i]表示对应的平方数的最小个数
    • 那么i如果是一个完全平方数，那么dp[i]=1
    • 否则，就从剩下的的dp[i- qqrt*sqrt]+1
    • 其实就是零钱兑换，找到所有的小于目标值的平方数，然后怎样取最小，
    • 假如是12，下面的平方数有1，2，3，减去之后对应11，9，8，我们只要看11，9，8谁对应的最小即可

2. DFS

深度优先搜索主要用于寻找可行解

2.1 岛屿的最大面积(695)

  • 思路：
    首先我们的每一步都有四种走法，我们可以将其先写出来{(1,0),(0,1),(-1,0),(0,-1)}

我们的dfs算法主要实现的是对于任意一个给定的节点，求出它的面积，然后遍历所有的节点从而求得最大面积，可以利用一个布尔数组实现剪枝

2.2 岛屿数量(200)

  • 思路：

    • 我们刚才已经能找到所有的岛屿了，也能比较出其最大面积了，其实只需要修改一点，判断当前遍历到的节点面积是否为0即可

2.3 被围绕的领域（130）

  • 思路：

    • 我们刚才已经能够找到所有的O领域了，在每一次找寻的时候，我们可以将其存储起来，返回一个布尔值，代表是否与边界相连，如果不相连的话，就把他再主函数中重置，否则，将集合消除

    这里的思路是先将四周的区域用另一个字符填充，然后再填充所有的其他领域
