0-1背包

一个容量为 N 的背包，要用这个背包装下物品的价值最大，这些物品有两个属性：体积 w 和价值 v。

定义一个二维数组 dp 存储最大价值，其中 dp[i][j]表示前 i 件物品体积不超过 j 的情况下能达到的最大价值。设第 i 件物品体积为 w，价值为 v，根据第 i 件物品是否添加到背包中，可以分两种情况讨论：

  • 第 i 件物品没添加到背包，总体积不超过 j 的前 i 件物品的最大价值就是总体积不超过 j 的前 i-1 件物品的最大价值，dp[i][j] = dp[i-1][j]。
  • 第 i 件物品添加到背包中，dp[i][j] = dp[i-1][j-w] + v。

空间优化

在程序实现时可以对 0-1 背包做优化。观察状态转移方程可以知道，前 i 件物品的状态仅与前 i-1 件物品的状态有关，因此可以将 dp 定义为一维数组，其中 dp[j] 既可以表示 dp[i-1][j] 也可以表示 dp[i][j]。此时，

因为 dp[j-w] 表示 dp[i-1][j-w]，因此不能先求 dp[i][j-w]，防止将 dp[i-1][j-w] 覆盖。也就是说要先计算 dp[i][j] 再计算 dp[i][j-w]，在程序实现时需要按倒序来循环求解。

1.1 分割等和子集

  • 思路：

    • 我们换一种想法，先求出总和，然后如果这个和是奇数，自然是不可能的，如果是偶数，那就相当于我们在这样的一个数组中寻找，能不能找到和为target的一种组合，且每个元素只能选择一次。

  • 思路2：

    • 我们想一下背包问题的本质是什么？本质问题是，一群数字，我们每个数字只有选择和不选择两种情况，并且会有一个容量的限制，这一题也是一样。我们要判断的是给定数组中能否找出一种组合，其加和结果是sum/2
    • 设定dp数组，其中dp[i][j]代表的是只使用前i个数字，能够找到一种组合其和为j，最后我们要求的结果自然就是dp[len+1][sum/2]
    • 对于第i+1个元素，无非是选择与不选择两种情况
      • 假如nums[i] == sum/2，直接返回true
      • 假如不选择nums[i],结果自然就是dp[i-1][j]
      • 假如选择了nums[i],那么结果就是dp[i-1][j-nums[i]]

1.2 目标和

  • 思路1： DFS

    • 既然每个数字只有两种选择情况，我们可以假设成一棵二叉树，从0开始向下搜索，直到找到和为target
    • 不过这里我们搜索的时候，不需要使用for循环，因为我们当前要添加的数字就是当前这一个

  • 思路2：

    • 切换成01背包问题
    • 设置取正的和为sum，取－的和为miu，即sum - miu = target,又有sum+ miu = arr_Sum,所以sum = (arr_sum+ targer)/2，不过要先行判断奇偶

1.3 一和零

  • 思路：

    • 这里0和1的数量即相当于限制我们背包的两个维度，设置dp[i][j][k]为使用前i个元素的情况下，使用j个0，k个1，能拼出来的最大个数

完全背包

完全背包与01背包的区别在于这里我们每个物品可以选择无限次，只是变成三重循环即可。

dp[i][j] = max(dp[i-1][j-w[i]*k]+v[i]*k, dp[i][j]);
