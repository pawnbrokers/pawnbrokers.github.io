1. 思考的步骤

动态规划一般对应的是求最值，然后问题的解决依赖于子问题的解决，并且子问题是重叠的可递归的子问题，还有就是动态规划问题一定具有“最优子结构”，这样我们才可以通过子问题的最值来得到原问题的最值。

三要素：

  • 重叠子问题
  • 最优子结构
  • 状态转移方程

思考步骤：

  • 明确状态
  • 定义dp[]数组或者函数的含义
  • 明确选择
  • 明确base case

2. 打家劫舍

  • 思路：

    • 假设dp[i]为对应下标位置处的最大抢劫金额，那么它主要有两种可能
      • 当前这个也抢，也就是dp[i-2]+nums[i]
      • 当前这个不抢，也就是dp[i-1]

3. 打家劫舍II

  • 思路：

    • 我们可以在原有的数组上，那么我们能偷窃到的最大金额无非是两种情况
      • 前面的原有0-n-1个元素（因为末尾是不能选择的）
      • 后面的1-n的元素（第一个是不可以选择的）

    这里不需要纠结假如第一种情况没选第一个元素怎么办？即这样意味着我们的最有一个元素是实际上可选的。

    这一类情况自然已经被我们的第二种情况包含在内了。

4. 矩阵路径

4.1 矩阵的最小路径和

  • 思路：

    • 这一类问题很直接，我们当前位置设为（i,j）那么dp[i][j]代表以当前坐标为右下角所绘制出来的矩形的路径总和最小值，它只能是从上面来或者下面来

4.2 不同路径

  • 思路：

    • 设置dp数组，dp[i][j]代表当前下标下的路径数目，那么dp[i][j]=dp[i][j-1]+ dp[i-1][j]

5. 数组区间

5.1 数组区间和

  • 思路：

  • 如果是会多次调用此方法的话，我们可以将其先存储起来

5.2 数组中等差递增子区间的个数（413）

  • 思路:

    • 要求p+1<Q，即等差数列至少要有三个数字组成

    • 设置dp数组，dp[i]表示以当前下标结尾的等差数列的个数，那么如果num[i] - nums[i-1] == nums[i-1] - nums[i-1]，dp[i] = dp[i-1]+1,否则当前就是0

6. 分割整数

6.1 分割整数的最大乘积(其实就是剪绳子)

  • 思路：

    • 一定要更多地分成2和3，其中3的优先级更高

6.2 完全平方数

  • 思路：

    • 其实就是零钱问题，我们找到小于n的所有平方数，然后在里面进行选择即可

6.3 解码方法（91）

  • 思路：

    • 设dp[i]为对应下下标对应的解码方法数目，dp[i+1]有三类情况

        1. s[i+1] =0?

          如果s[i] =1或者2，则dp[i+1] = dp[i-1]，负责return 0；

        1. 如果s[i] =1,那么dp[i+1] = dp[i-1] + dp[i-2]
        1. 如果s[i] =2,并且s[i+1] 在1和6之间，则同上
